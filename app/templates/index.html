Requirements

The API client or API server must comply with the following requirements.

Authentication

Requests must include an AUTH_TOKEN header. This token will have been provided to you alongside the initial reference to this webserver. Client requests to authenticated endpoints (i.e., every endpoint except this one) without a valid token will return a 401 (Unauthorized) response.

API End Points

This webserver hosts a single API endpoint, /v1/user. The endpoint is scoped with a user_id, e.g., /v1/user?user_id=INT. The following methods are supported: POST, PUT, GET, DELETE. Requests are scoped with a valid user_id, with the exception of POST, which returns the user_id in the request response object.

Successful POST, PUT, and GET responses will be of the form 

{'users': [{"user_id": INT, "username": STRING, "password": SHA1(USERNAME+PASSWORD)}], 'errors': []}

DELETE requests will be of the form 

{'users': [], 'errors': []}

POST

Create a user. The POST body must be of the form 

{"username": "STRING", "password": "STRING"}

PUT

Update a user. The PUT body must be of the form 

{"username": "STRING", "password": "STRING"}

GET

Return a user. If the user_id parameter is omitted, all users are returned.

DELETE

Delete a user.
